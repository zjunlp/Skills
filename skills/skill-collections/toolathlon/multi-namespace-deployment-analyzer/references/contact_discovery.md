# Contact Discovery Strategies

## Primary Methods for Finding Cluster Administrators

### 1. Email Search Patterns
Search emails using these keywords in order:
- "cluster management"
- "cluster admin"
- "kubernetes"
- "infrastructure"
- "devops"
- "platform team"

### 2. File System Discovery
Check these common locations for contact information:
- `/workspace/dumps/workspace/` - Look for config files, documentation
- `/shared/configs/` - Kubernetes configs may contain contacts
- `/etc/` - System configuration files
- Home directories - User-specific contact info

### 3. Document Analysis
Examine these document types:
- **PDFs**: Organizational charts, responsibility matrices, guidelines
- **Markdown/Text files**: READMEs, documentation, runbooks
- **YAML/JSON configs**: Application configurations, team manifests
- **Spreadsheets**: Team rosters, contact lists

### 4. Common File Patterns
Search for files containing:
- `contact`
- `team`
- `admin`
- `responsibility`
- `oncall`
- `roster`

### 5. Fallback Contacts
If specific cluster admin not found:
- Use team distribution lists
- Contact infrastructure team lead
- Use general support email
- Escalate to department head
